01 — Vision & contexte

Qu'est-ce que Passeport Maison ?

Application mobile permettant aux propriétaires de documenter, valoriser et partager leur bien immobilier. Elle centralise le plan 2D/3D, les équipements, les matériaux et génère une fiche technique partageable.

Le problème résolu
Quand on vend ou loue un bien, il est impossible de prouver la valeur de ses équipements et rénovations. Les acheteurs ne savent pas ce qu'ils achètent, les propriétaires ne peuvent pas justifier leur prix.
La solution
Un "carnet d'entretien numérique" avec plan interactif, inventaire des équipements et fiche technique partageable selon le profil du destinataire (acheteur, locataire, artisan).
Cible principale
Propriétaires qui envisagent de vendre ou louer leur bien dans les 6 à 24 mois.
Cible secondaire
Artisans et fournisseurs qui souhaitent promouvoir leurs services auprès de propriétaires qualifiés.
Modèle économique
Freemium : accès gratuit limité. Offre Pro débloque le coffre-fort documents, le rendu 3D réaliste et les fonctionnalités avancées.
02 — Périmètre V1

Ce qui est dans scope

Le périmètre est défini par les 32 écrans de DOC/wireframes.html. Tout ce qui n'y figure pas est hors scope V1.

✅ Dans le scope V1
  • Authentification magic link (sans mot de passe)
  • Onboarding 5 slides + création du logement
  • Éditeur de plan 2D/3D (open3dFloorplan intégré)
  • Saisie rapide des pièces avec compteurs et dimensions
  • Inventaire des équipements : IA photo, scan code-barre, saisie manuelle
  • Deux types d'équipement : Fixe (valorisation) et Mobilier (vente)
  • Dashboard : valeur documentée du bien + miniature plan
  • Galerie photo regroupée par pièce
  • Fiche technique : partage multi-profil (acheteur, locataire, artisan)
  • Export PDF / JSON / DXF
  • Marketplace : annonces particuliers + catalogue pros
  • Mise en vente d'un équipement mobilier depuis le passeport
  • Messagerie temps réel (SSE) acheteur ↔ vendeur
  • Annuaire artisans & fournisseurs avec filtre géographique
  • Profil utilisateur + gestion des annonces actives
  • Freemium : mur payant + upgrade modal
  • Coffre-fort documents (Pro)
  • Rendu 3D réaliste (Pro)
  • Espace Pro artisan : inscription, validation, dashboard, catalogue
🚫 Hors scope V1 (ne pas implémenter)
  • Estimation / devis travaux de construction ou de rénovation
  • Paiement en ligne pour les transactions marketplace
  • Notifications push (uniquement badge dans l'app)
  • Application multi-logements (V1 = 1 logement par compte)
  • Synchronisation Apple RoomPlan (import prévu mais non fonctionnel en V1)
  • Mode démo (remplacé par l'onboarding)

51 user stories · 11 epics

Chaque story suit le format : "En tant que [persona], je peux [action] afin de [bénéfice]."

🔐
Epic 1 — Authentification
3 stories
  • US-01En tant que visiteur, je peux créer un compte avec mon adresse email (sans mot de passe) pour accéder à l'application. Un lien de connexion unique valable 15 minutes m'est envoyé par email.
  • US-02En tant qu'utilisateur existant, je peux me reconnecter via un nouveau magic link si ma session a expiré.
  • US-03En tant qu'utilisateur, je peux me déconnecter depuis mon écran Profil pour fermer ma session de manière sécurisée.
🏗️
Epic 2 — Onboarding & Création du bien
5 stories
  • US-04En tant que nouvel utilisateur, je vois un onboarding de 5 slides qui présente les fonctionnalités clés (plan, équipements, valorisation, marketplace, partage). Je peux le passer à tout moment.
  • US-05En tant qu'utilisateur, je peux créer mon logement en renseignant son nom, son type (maison, appartement, bureau…), le nombre d'étages, la surface, l'année de construction et une adresse optionnelle.

  • US-07En tant qu'utilisateur en saisie rapide, je peux ajuster le nombre de pièces par type (salon, chambre, cuisine, salle de bain, WC, bureau, couloir, garage…) avec des compteurs + / −.
  • US-08En tant qu'utilisateur en saisie rapide, je peux configurer les dimensions (largeur × longueur) et le nom de chaque pièce, avec calcul automatique de la surface en m².

Epic 3 — Éditeur de plan
5 stories

  • US-21En tant qu'utilisateur, je peux modifier les informations d'un équipement ou le supprimer depuis sa vue détail.

07 — Stack technique

Architecture full-stack SvelteKit

Application mobile-first. Référence de design : iPhone 390 × 844 px. Toutes les pages sont des routes SvelteKit — pas d'app séparée front/back.

Framework
SvelteKit 2.5 + Svelte 5
Full-stack. Routing par fichiers. +page.svelte + +server.ts. SSR activé.
Base de données
PostgreSQL + Prisma ORM
Prisma Client généré. Migrations avec prisma migrate. Schema dans prisma/schema.prisma.
Authentification
Magic link · Resend · JWT
Pas de mot de passe. Lien unique valable 15 min envoyé par email via Resend. Session JWT dans cookie httpOnly.
Intelligence artificielle
Anthropic SDK (Vision)
Analyse photo d'équipement. Retourne nom, marque, modèle, catégorie, prix de remplacement estimé, score de confiance.
Éditeur de plan
open3dFloorplan (Three.js)
Intégré dans la route /editor. Dessin vectoriel 2D, rendu 3D temps réel, 245+ équipements pré-définis.
Temps réel
Server-Sent Events (SSE)
Pas de WebSocket. Endpoint GET /api/conversations/[id]/stream. Unidirectionnel serveur → client, suffisant pour la messagerie.
State management
Svelte stores
Runes Svelte 5 ($state, $derived) pour le state local. Svelte stores pour le state global (user, currentHouse).
Emails transactionnels
Resend
Magic link uniquement en V1. Template HTML simple. Domaine expéditeur à configurer.
Référentiel de départ : Le projet est basé sur open3dFloorplan-main. L'éditeur est déjà fonctionnel. Toute la couche SvelteKit (auth, API, BDD) est à construire par-dessus.

11 — Temps réel

Messagerie SSE (Server-Sent Events)

Pas de WebSocket. L'app utilise SSE (unidirectionnel serveur → client). Pour envoyer, le client fait un POST classique. Pour recevoir en temps réel, il s'abonne au stream GET.

Implémentation côté serveur (SvelteKit +server.ts) :

// src/routes/api/conversations/[id]/stream/+server.ts
export const GET: RequestHandler = ({ params, locals }) => {
  const { id } = params
  const stream = new ReadableStream({
    start(controller) {
      // Poll DB toutes les 2s ou utiliser pg LISTEN/NOTIFY
      const interval = setInterval(async () => {
        const msgs = await getNewMessages(id, lastSeenAt)
        msgs.forEach(m => {
          controller.enqueue(`data: ${JSON.stringify(m)}\n\n`)
        })
      }, 2000)
      return () => clearInterval(interval)
    }
  })
  return new Response(stream, {
    headers: { 'Content-Type': 'text/event-stream', 'Cache-Control': 'no-cache' }
  })
}

Connexion côté client (Svelte) :

// Dans le composant conversation
let messages = $state([])
onMount(() => {
  const es = new EventSource(`/api/conversations/${id}/stream`)
  es.onmessage = (e) => {
    messages = [...messages, JSON.parse(e.data)]
  }
  return () => es.close()
})
12 — Hors périmètre

Ne pas implémenter en V1

Estimation / Devis construction
Aucun module de calcul de coût de travaux, d'estimation de rénovation ou de devis construction. Ce périmètre est explicitement exclu de V1.
Paiement en ligne
V1 = listing + chat uniquement. La transaction se fait hors app. Pas de Stripe, pas de paiement intégré.
Multi-logements
Un seul logement par compte en V1. La contrainte @unique sur userId dans House le garantit côté base.
Notifications push
Uniquement un badge dans l'app (unread count). Pas de push notifications système, pas de service worker.
